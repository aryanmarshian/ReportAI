# ReportAI 📊

**ReportAI** is an autonomous investment analysis platform powered by intelligent agents and LLMs. It orchestrates multiple AI agents to analyze investments, extract key insights, develop strategic plans, and generate comprehensive investment research reports.

## Features ✨

- **Multi-Agent Architecture**: Coordinated AI agents for specialized analysis tasks
- **Autonomous Orchestration**: Intelligent task pipeline execution with state management
- **LLM-Powered Analysis**: Leverages GPT models for deep financial analysis
- **Real-time Task Monitoring**: Track orchestration state and agent outputs
- **Professional Report Generation**: Converts structured analysis into polished investment documents
- **RESTful API**: Clean HTTP endpoints for task creation and status monitoring
- **Web Console**: Interactive UI for submitting analyses and inspecting results
- **Async-First Architecture**: Built with FastAPI for high performance

## Installation 📦

### Prerequisites
- Python 3.9+
- PostgreSQL database
- OpenAI API key

### Setup

1. **Clone the repository**
   ```bash
   cd reportai
   ```

2. **Create virtual environment**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

4. **Configure environment**
   Create a `.env` file in the root directory:
   ```
   OPENAI_API_KEY=your_api_key_here
   DATABASE_URL=postgresql+asyncpg://user:password@localhost/reportai
   ```

5. **Initialize database**
   ```bash
   # Run the schema from app/db/schema.sql
   psql -U your_user -d reportai -f app/db/schema.sql
   ```

## Quick Start 🚀

1. **Start the server**
   ```bash
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

2. **Open the web console**
   Navigate to `http://localhost:8000/static/index.html`

3. **Submit an analysis**
   - Enter a User ID
   - Provide your analysis prompt (e.g., "Analyze Tesla as a long-term investment")
   - Click "Launch Analysis"
   - Monitor the task state and view results

## API Endpoints 🔌

### POST `/analyze`
Submit a new analysis task
```json
{
  "user_id": 1,
  "input_text": "Analyze this investment opportunity..."
}
```

**Response:**
```json
{
  "task_id": 123,
  "status": "pending"
}
```

### GET `/status/{task_id}`
Retrieve current task status
```json
{
  "status": "completed"
}
```

### GET `/plan/{task_id}`
Get the task plan generated by the planner agent
```json
{
  "task_id": 123,
  "status": "completed",
  "plan": { "steps": [...] }
}
```

### GET `/report/{task_id}`
Download the generated report as a file

## Agent System 🤖

ReportAI uses a modular agent system:

1. **Planner Agent** - Analyzes the input and creates an analysis plan
2. **Extractor Agent** - Extracts relevant information and facts
3. **Report Writer Agent** - Synthesizes outputs into a professional report

Each agent communicates through structured JSON schemas, ensuring data consistency throughout the pipeline.

## Development 👨‍💻

### Adding a New Agent

1. Create a new file in `app/agents/`
2. Implement agent logic following the pattern in existing agents
3. Register in the task pipeline (`app/services/task_runner.py`)

### Running in Development

```bash
# With auto-reload
uvicorn app.main:app --reload

# With debug logging
DEBUG=True uvicorn app.main:app --reload
```

## Performance Considerations ⚡

- **Async/Await**: All I/O operations are async for optimal throughput
- **Connection Pooling**: SQLAlchemy manages database connections efficiently
- **LLM Caching**: Consider implementing response caching for repeated analyses
- **Background Tasks**: Long-running reports generated asynchronously

## Troubleshooting 🔧

### Database Connection Errors
- Verify PostgreSQL is running
- Check DATABASE_URL in environment
- Ensure database exists: `createdb reportai`

### OpenAI API Errors
- Verify API key is correct
- Check API rate limits
- Ensure account has sufficient credits

### Frontend Not Loading
- Confirm server is running on port 8000
- Check browser console for network errors
- Clear browser cache
